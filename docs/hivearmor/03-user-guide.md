## 1. Logging In

1. Open your browser and go to `https://<SERVER_NAME>/login`
2. Enter your username and password
3. If Two-Factor Authentication is enabled, open your authenticator app and enter the 6-digit code

### Session expiry

Sessions expire after 24 hours (or 30 days if "Remember Me" was checked). You will be redirected to the login page automatically.

### Forgotten password

Click **Forgot Password** on the login page → enter your email → check your inbox for a reset link.

## 2. Dashboard Overview

The dashboard is your home screen. It provides a real-time summary of platform activity:

| Widget | Description |
|---|---|
| Alert Severity Distribution | Pie/bar chart of alerts by severity over the selected time range |
| Active Alerts | Count of open alerts by status |
| Top Attackers | Source IPs with the most correlated alerts |
| Top Targets | Destination IPs / hostnames most frequently targeted |
| Events Over Time | Timeline of events ingested per hour |
| Agent Health | Count of online vs. offline agents |
| Compliance Score | Quick summary of compliance control pass rates |

### Adjusting the time range

Use the **time picker** in the top-right corner. Options: Last 15 minutes, 1 hour, 24 hours, 7 days, 30 days, custom range.

## 3. Alerts

Alerts are the primary work unit for SOC analysts. They are generated by the correlation engine when log events match a rule.

### Navigating to Alerts

Click **Alerts** in the left navigation.

### Alert list view

| Column | Description |
|---|---|
| Alert Name | The rule that triggered the alert |
| Severity | Critical / High / Medium / Low |
| Status | Open / In Review / Closed / Ignored |
| Time | When the alert was first created |
| Last Echo | When the alert was last updated |
| Sensor | Which agent/source sent the triggering log |
| Echoes | How many times this pattern has repeated |
| Target | Destination IP, hostname, or user |
| Adversary | Source IP, hostname, or user |

### Filtering alerts

**Left sidebar filters:**
- Free text search across all alert fields
- **Severity** checkboxes: Critical, High, Medium, Low
- **Status** tabs: All | Open | In Review | Closed
- **Category** (MITRE ATT&CK tactic), **Sensor**, **Tags**
- **Adversary IP** / **Target IP** with autocomplete

**Time range picker** (top right) applies to all filters.

### Working an alert

Click any alert row to open the detail panel on the right:

#### Tab 1 — Detail
Shows full alert metadata:
- Description and references
- MITRE ATT&CK technique (e.g., T1110 — Brute Force)
- Target and adversary panels with IP, hostname, user, ASN
- All raw log fields

#### Tab 2 — SOC AI
Click **Analyze with AI** to get:
- Classification (True Positive / False Positive / Needs Investigation)
- Reasoning
- Recommended next steps

#### Tab 3 — Events Related
Raw log events that triggered this alert. Click any event to see the full structured fields.

#### Tab 4 — Incident Detail
If the alert is linked to an incident, shows incident metadata and link.

#### Tab 5 — Map
Geographic visualization of source/destination IPs.

#### Tab 6 — Echoes
Timeline of related alerts (same pattern, different instances). Useful for spotting campaigns.

#### Tab 7 — Alert History
Audit log of status changes and analyst comments.

#### Tab 8 — Rules Applied
Which auto-tag rules matched this alert.

### Changing alert status

**Single alert:**
1. Open the alert detail panel
2. Click the current status badge
3. Select: Open / In Review / Completed / Ignored
4. For Completed and Ignored: enter an observation note (required) and optionally create a False Positive rule

**Bulk status change:**
1. Check multiple alerts in the list
2. Click **Change Status** in the bulk action bar at the top
3. Select the new status

### Creating an incident

**From a single alert:**
1. Open the alert detail panel
2. Click **Create Incident** or **Add to Incident**

**From multiple alerts:**
1. Select alerts with checkboxes
2. Click **Create/Add to Incident** in the bulk action bar

### Adding comments

1. Open the alert detail panel
2. Scroll to the **Comments** section
3. Type your note and click **Add**

## 4. Log Analyzer (Threat Hunting)

The Log Analyzer lets you search raw log events directly. Use it for threat hunting, incident investigation, and ad-hoc queries.

Navigate to **Log Analyzer** in the left navigation.

### Running a search

1. Set the **time range** (default: last 24 hours)
2. Select **Log Type** from the dropdown (e.g., `linux`, `wineventlog`, `paloalto`)
3. Type a query in the search bar using KQL (Kibana Query Language):
   - `log.message: "Failed password"` — full text search in a field
   - `origin.ip: 10.0.0.5` — exact value match
   - `severity: high AND origin.ip: 10.*` — combined conditions
   - `log.eventId: 4625` — Windows event ID
4. Click **Search**

### Pivoting from an alert

From an alert's "Events Related" tab, click any field value to pivot into Log Analyzer pre-filtered to that value.

### Saving queries

Click **Save Query** after running a search to save it for later. Saved queries appear in the left sidebar under **Saved Queries**.

### Exporting results

Click **Export** (CSV or JSON) to download search results.

## 5. Incidents

Incidents group related alerts into a single investigation case.

Navigate to **Incidents** in the left navigation.

### Incident list

| Column | Description |
|---|---|
| Name | Incident title |
| Severity | Severity assigned by the creator |
| Status | Open / In Review / Resolved / Closed |
| Assignee | Analyst responsible |
| Alerts | Number of linked alerts |
| Created | When the incident was created |

### Incident detail view

Click an incident to open it:
- **Summary:** title, description, timeline, severity, MITRE tactics
- **Alerts:** list of all linked alerts (click to view, unlink, or add new)
- **Notes:** analyst notes and investigation log
- **Activity:** change history

### Updating incident status

1. Click the status badge at the top of the incident
2. Select new status
3. Enter a note (optional but recommended)
4. Click **Save**

## 6. Compliance

Navigate to **Compliance** for framework-based control evaluations.

### Dashboard

Shows a compliance score card for each enabled framework (HIPAA, PCI-DSS, ISO 27001, NIST CSF, SOC 2).

### Control detail

Click any framework → see each control listed with:
- **Status:** Pass / Fail / Not Applicable
- **Evidence:** Log data or configuration that supports the control
- **Last evaluated:** Timestamp of last check

### Generating a compliance report

1. Navigate to **Compliance → Reports**
2. Select the framework
3. Select date range
4. Click **Generate PDF**

The PDF report is generated by the `web-pdf` service and includes control pass/fail details, evidence references, and an executive summary.

## 7. Vulnerability Scanner

Navigate to **Vulnerability Scanner** to run and view network vulnerability scans.

### Running a scan

1. Click **New Scan**
2. Enter the target IP or CIDR range (e.g., `192.168.1.0/24`)
3. Select scan type: Quick / Full
4. Click **Start**

Scan results appear when complete, showing:
- Open ports
- Identified services and versions
- Known CVEs (from the threat intelligence feeds)

---

## 8. Asset Discovery

Navigate to **Asset Discovery** for automatic network asset inventory.

The scanner periodically discovers hosts on your configured network ranges and enriches them with:
- Hostname (reverse DNS)
- OS fingerprint
- Open services
- Last seen timestamp

Click any asset to view its history and linked alerts.

## 9. Reports

Navigate to **Reports** to create and manage security reports.

### Types

| Report Type | Description |
|---|---|
| Alert Summary | Count and trend of alerts by severity, category, sensor |
| Compliance | Framework-specific control evaluations |
| Incident Summary | Open/closed incidents over time |

### Creating a one-time report

1. Click **New Report**
2. Select type and configure parameters
3. Click **Generate**

### Scheduling a report

1. Click **New Scheduled Report**
2. Configure: name, type, schedule (daily/weekly/monthly), recipients (email)
3. Click **Save**

## 10. SOC AI Assistant

The SOC AI is embedded throughout the platform and is powered by a language model running inside the `soc-ai` plugin within the event processor.

### Where it appears

- **Alert detail → SOC AI tab:** Analyzes a specific alert
- **Incident detail → AI Assist button:** Summarizes the investigation and suggests next steps
- **Log Analyzer → AI Query button:** Translates natural language into a log query

### Using the alert AI analysis

1. Open any alert
2. Click the **SOC AI** tab
3. Click **Analyze**
4. Review:
   - **Classification** — likely true positive, false positive, or needs investigation
   - **Reasoning** — explanation of the classification
   - **Next Steps** — recommended investigation or response actions

## 11. User Profile & 2FA Setup

### Accessing your profile

Click your avatar or username in the top-right corner → **Profile**.

### Setting up Two-Factor Authentication

1. Navigate to **Profile → Two-Factor Authentication**
2. Click **Enable**
3. Scan the QR code with your authenticator app (Google Authenticator, Authy, 1Password, Bitwarden)
4. Enter the 6-digit verification code to confirm
5. Save the backup codes in a secure location

### Changing your password

1. Navigate to **Profile → Change Password**
2. Enter current password and the new password twice
3. Click **Save**

## 12. Keyboard Shortcuts & Tips

| Shortcut | Action |
|---|---|
| `/` | Focus the search bar |
| `Esc` | Close side panel |
| `↑` / `↓` | Navigate alert list |
| `Enter` | Open selected alert |
| `Shift+Click` | Select range of alerts |

### Tips

- **Time range matters:** All views respect the time picker. If you're not seeing expected alerts, extend the range.
- **Pivot from fields:** In alert detail and log analyzer, click any field value to filter or search on it.
- **Use the SOC AI early:** Running AI analysis on ambiguous alerts saves investigation time.
- **Echoes show campaigns:** An alert with many echoes indicates repeated behavior — often an active attack or misconfiguration.
- **False Positive rules:** When dismissing an alert, create a False Positive rule to suppress similar alerts in the future. Review periodically.
